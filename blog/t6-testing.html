May 26th - Scope, and how it works in Javascript

Scope is a way of describing how a variable can be used. There are two kinds of scope: Global and Local.

A variable has Global scope if it is declared outside of a function: 

var globalVar = 123;

function myFunc () {
  if (globalVar = 123) {
  alert("Woohoo!");
  }
}

If it's global, it can be accessed from anywhere in the code. On one hand this is very convenient. On the other, it can be a liability, especially when the code is very complex. This is because it can be hard to keep track of which bits of the code are using the variable. Multiple parts trying to use it concurrently can cause conflicts and the code will not work, or it will behave in a strange manner.

Local scope is held by a variable that is declared inside a function. 

function timesTwo (number) {
  var localVar = number * 2;
  alert("The total is " + localVar);
  }

timesTwo(7);

If a variable is local it is only accessible inside of that function. This avoids conflicts like that described above, but it also means that a potentially useful variable can't be used elsewhere.

Variables that are declared inside a "nested" function (that's a function inside another function) are local only to the function they reside in. They are not accessible by the 'grandparent' function.

Hoisting is a concept that is affected by scope. It means that a variable can declared below code of the same scope where it has already been used - it is as if the variable was declared at the very top of the code and is used afterward. This is important because code is executed from top to bottom. 

x = 5; // Assign 5 to x

elem = document.getElementById("demo"); // Find an element 
elem.innerHTML = x;                     // Display x in the element

var x; // Declare x

Hoisting only affects variable declarations, not initialisations. In the example above, the variable is being declared at the bottom and initialised (given a value) at the top. In the example below, the variable y is initialised below the code: this will not work because the code cannot know the value of y: 

var x = 5; // Initialize x

elem = document.getElementById("demo"); // Find an element 
elem.innerHTML = x + " " + y;           // Display x and y

var y = 7; // Initialize y

Hoisting is often overlooked by developers and can lead to bugs in software. It is best practice to always declare variables at the beginning of code for this reason.
